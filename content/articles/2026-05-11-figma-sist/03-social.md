# Social posts — Figma first is killing your agentic design workflow

## LinkedIn

𝗙𝗶𝗴𝗺𝗮 𝗳𝗶𝗿𝘀𝘁 𝗿𝗲𝗳𝗹𝗲𝘅 𝗶𝘀 𝗸𝗶𝗹𝗹𝗶𝗻𝗴 𝘆𝗼𝘂𝗿 𝗮𝗴𝗲𝗻𝘁𝗶𝗰 𝗱𝗲𝘀𝗶𝗴𝗻 𝘄𝗼𝗿𝗸𝗳𝗹𝗼𝘄!

Most designers open Figma as a reflex when they need to understand what they're building. However, the days when "adding all screens in Figma" was a reasonable workflow are gone.

The classic design flow went something like this:

Project plan → Figma → Development → add everything else → release

That was fine when developers interpreted designs manually. It's a problem when agents build them. Agents don't read Figma files the way developers do. They need dense context — purpose, behavior, content, error states. Figma gives them pixels.

𝘠𝘰𝘶 𝘥𝘳𝘢𝘸 𝘵𝘩𝘦 𝘱𝘳𝘰𝘥𝘶𝘤𝘵, 𝘵𝘩𝘦 𝘢𝘨𝘦𝘯𝘵𝘴 𝘢𝘳𝘦 𝘴𝘵𝘪𝘭𝘭 𝘪𝘨𝘯𝘰𝘳𝘢𝘯𝘵.

Pixels without context = vibe coding flames. An endless loop of prompts that makes your project worse with each iteration.

The fix isn't AI drawing for you in Figma. That just gets you lost faster.

𝗧𝗵𝗲 𝘀𝗵𝗶𝗳𝘁: build context, design in code and use Figma for visual adjustments:

→ 𝗦𝘁𝗿𝗮𝘁𝗲𝗴𝘆 — what are you building and why?
→ 𝗦𝗰𝗲𝗻𝗮𝗿𝗶𝗼𝘀 — wireframe what it should actually do, step by step
→ 𝗦𝗽𝗲𝗰𝗶𝗳𝗶𝗰𝗮𝘁𝗶𝗼𝗻𝘀 — here is where your vision becomes real for the agents
→ 𝗜𝗺𝗽𝗹𝗲𝗺𝗲𝗻𝘁𝗮𝘁𝗶𝗼𝗻 — let agents build from real specs
→ 𝗙𝗶𝗴𝗺𝗮 𝗹𝗮𝘀𝘁 — open the code for visual refinement

This is exactly the workflow in 𝗪𝗵𝗶𝘁𝗲𝗽𝗼𝗿𝘁 𝗗𝗲𝘀𝗶𝗴𝗻 𝗦𝘁𝘂𝗱𝗶𝗼, the official UX module for the BMad method. It is free and open source for every designer on the planet!

A complete philosophy for designers to be effective with agents. Strategy, scenarios, agentic development — with a full curriculum to get you there.
