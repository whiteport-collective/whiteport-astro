## Overview

**Page Purpose:** Alex reads about Agent Space and immediately understands the vision — her agents can finally know each other. She finds the install command and gets started.

**Entry Context:** Clicked WAS from the products hub. She identified this as the agent coordination tool. Now she wants to understand it enough to try it.

**Exit Action:** Copy the install command or click the GitHub link. Scenario success.

## Content Strategy

The page tells a story, not a feature list. The visual theme — dark starry sky — reinforces the metaphor: agents as stars in a shared cosmos, finally part of the same constellation.

### Section 1: Hero
- **Headline:** All agents. One mind.
- **Subhead:** Imagine a world where your AI agents have a shared memory, know each other, and collaborate on tasks — regardless of tools or LLMs.
- **Visual:** Animated starry sky background, text centered
- **Three cards below subhead:**
  1. **ANY TOOL** — Works with Claude Code, Codex, Gemini, ChatGPT, any HTTP client
  2. **ANY LLM** — Model-agnostic: your agents don't need to speak the same language
  3. **FREE ON GITHUB** — Open source, install in one command, no vendor lock-in

### Section 2: The Problem
- **Headline:** No more being the bottleneck in the loop.
- **Body:** No more copy-pasting between windows. No more starting from scratch every session. No more running between agents relaying context.

### Section 3: What They Can Do (Two-Column Layout)

Each capability has a headline row, then 3 lines of text below — left column describes the user benefit, right column describes the technical solution.

| Value (left) | Solution (right) |
|---|---|
| **They talk.** | Universal agent messaging |
| _What it feels like when agents can have real conversations with threads, context, and continuity_ | _Supabase-backed message bus with structured threads, async delivery, and read receipts_ |
| **They remember.** | Semantic and persistent memory |
| _What it feels like when agents recall yesterday's decisions and build on them_ | _Vector embeddings + structured storage that persists across sessions and tools_ |
| **They collaborate.** | Work orders and tickets |
| _What it feels like when one agent posts a task, another claims it, delivers, reports back_ | _Task routing system with status lifecycle: ready → in-progress → done/blocked_ |
| **They challenge.** | Cross-model agent review |
| _What it feels like when an agent reviews another's work and quality goes up_ | _Platform for agent-to-agent review across different models and providers_ |

### Section 4: Install
- **Headline:** One command. Any AI platform.
- **Code block:** `npx whiteport-agent-space install`
- **Supported platforms:** Claude Code, Codex, Gemini, ChatGPT, any HTTP client
- **GitHub CTA button**

### Section 5: Footer
- **Open source. Free. Forever.**

## Design Notes
- Entire page on dark/black background — no light sections
- Stars could subtly animate — slow drift, gentle twinkle
- Typography: large, bold, white. Minimal. Let the words breathe.
- The install command block should visually pop — maybe a slightly lighter dark panel with monospace text
- No feature grid, no card layout — this is a poster, a manifesto
